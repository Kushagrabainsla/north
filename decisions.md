Hierarchical Tool Selection: A Tree-Based Approach to Scalable AI Agent Decision Making
As AI agents grow more capable, giving them a flat list of all available tools starts to become a problem. It fills up the model's context window and makes it harder for the agent to pick the right tool as the number of tools grows. This paper explores a tree-based approach, where tools are organized from broad domains at the top down to specific tools at the leaf nodes. Instead of choosing from hundreds of tools at once, the agent makes a series of small, focused decisions as it walks down the tree. At each step, it only sees a small relevant subset of tools, which reduces confusion, improves accuracy, and leaves more context space for the actual task. We also propose building the tree dynamically at runtime, pruning irrelevant branches based on what the task is about before traversal even begins. This combines the structure of a fixed hierarchy with the flexibility of context-aware loading. Compared to existing approaches like RAG-based tool retrieval and sub-agent delegation, hierarchical tool selection is a simpler, more interpretable, and scalable solution for building production AI agent systems.
